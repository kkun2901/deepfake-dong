# 딥페이크 탐지 앱 프로젝트 개요

## 프로젝트 개요

### 제품명
딥페이크 탐지 앱 (Deepfake Detector App)

### 소개
딥페이크 탐지 앱은 Android 모바일 환경에서 실시간으로 딥페이크 영상을 탐지하고 분석하는 통합 애플리케이션입니다. React Native와 FastAPI 백엔드를 결합하여 구축되었으며, 플로팅 위젯 기반 스크린 레코딩과 딥러닝 모델 앙상블을 통한 정확한 딥페이크 탐지를 제공합니다.

이 앱의 가장 큰 특징은 사용자가 현재 사용 중인 앱 화면(예: 유튜브, 인스타그램, 틱톡 등)을 별도 앱 전환 없이 바로 녹화하고 분석할 수 있다는 점입니다. 의심스러운 영상을 발견하면 즉시 검증할 수 있으며, 분석 결과는 타임라인 세그먼트 기반으로 제공되어 영상 내 어느 구간에서 딥페이크가 탐지되었는지 상세하게 확인할 수 있습니다. 또한 분석 결과를 딥러닝 연구용 데이터셋(JSONL, CSV, JSON)으로 자동 변환하여 다운로드할 수 있어 연구자들에게도 유용한 도구입니다.

### 프로젝트 목표
이 프로젝트는 모바일 환경에서 실시간 딥페이크 탐지 서비스를 제공하는 것을 목표로 합니다. 즉시성, 정확성, 사용성, 확장성, 효율성을 핵심 가치로 하여 개발되었습니다. 두 개의 Vision Transformer 모델을 앙상블 방식으로 결합하여 높은 탐지 정확도를 달성했으며, 플로팅 위젯을 통해 앱 전환 없이 즉시 녹화 및 분석이 가능하도록 설계되었습니다. 분석 결과는 딥러닝 학습 데이터셋으로 자동 변환되어 연구 활용이 가능하며, 얼굴 감지 기반 스마트 프레임 추출 및 동적 프레임 샘플링으로 처리 속도를 최적화했습니다.

### 해결하고자 하는 문제
현재 소셜 미디어에서 딥페이크 영상이 빠르게 확산되고 있지만, 이를 신속하게 검증할 수 있는 도구가 부족한 상황입니다. 기존 딥페이크 탐지 도구는 주로 PC 기반으로 개발되어 있어 모바일 사용자가 즉시 사용하기 어렵습니다. 또한 의심스러운 영상을 발견했을 때 별도 앱으로 전환하거나 파일을 다운로드하는 과정에서 시간이 소모되어 정보 검증이 지연됩니다.

또한 딥페이크 탐지 모델 연구를 위한 구조화된 데이터셋 생성이 어렵고, 정적 화면이나 얼굴이 없는 영상에서의 오탐으로 인해 신뢰도가 저하되는 문제가 있습니다. 이 프로젝트는 이러한 문제들을 해결하기 위해 플로팅 위젯 기반 실시간 스크린 레코딩으로 즉시 녹화 및 분석이 가능하도록 했으며, 모바일 네이티브 앱으로 제공하여 언제 어디서나 접근 가능하도록 했습니다. 앱 전환 없이 현재 화면에서 바로 녹화를 시작할 수 있으며, 분석 결과를 자동으로 JSONL, CSV, JSON 형식의 데이터셋으로 변환하여 제공합니다. 얼굴 감지 기반 전처리 및 정적 화면 보정 알고리즘을 통해 오탐률을 감소시켰습니다.

### 프로젝트가 필요한 이유
딥페이크 기술의 발전으로 조작된 영상이 소셜 미디어를 통해 빠르게 확산되고 있습니다. 정치적 선동, 금전적 사기, 명예훼손 등 딥페이크 악용 사례가 증가하고 있어, 이를 즉시 검증할 수 있는 도구가 필수적입니다. 현재 사용자들의 대부분이 모바일 기기를 통해 콘텐츠를 소비하므로, 모바일 환경에 최적화된 탐지 도구가 필요합니다.

또한 딥페이크 영상의 확산 속도에 대응하기 위해 즉시 검증이 가능한 시스템이 필요하며, 딥페이크 탐지 모델의 지속적인 개선을 위한 고품질 데이터셋 생성 도구가 필요합니다. 일반 사용자도 쉽게 사용할 수 있는 도구를 제공하여 미디어 리터러시 교육 및 실천 도구로 활용할 수 있으며, 미디어 기업, 플랫폼 서비스, 보안 기업 등에 기술을 제공하거나 API로 제공할 수 있는 시장적 필요성도 있습니다.

### 목표 사용자
주요 사용자는 소셜 미디어를 활발히 사용하는 18-45세의 일반 소비자와 정보 검증에 관심이 있는 사용자, 딥페이크에 대한 우려가 있는 사용자입니다. 또한 뉴스 확인 및 팩트체크를 수행하는 미디어 전문가 및 저널리스트, 소셜 미디어 콘텐츠를 검증해야 하는 미디어 관계자도 목표 사용자에 포함됩니다.

부차적 사용자로는 딥페이크 탐지 모델 연구를 수행하는 연구자, 딥러닝 데이터셋이 필요한 연구자, 컴퓨터 비전 연구자 등이 있습니다. 또한 미디어 리터러시 교육을 수행하는 교사, 디지털 시민 교육 프로그램 운영자 등 교육 관계자와 콘텐츠 검증이 필요한 플랫폼 서비스 기업, 보안 및 인증 서비스를 제공하는 기업 등도 포함됩니다.

## 핵심 기능

### Must: 필수 기능
백엔드 측면에서는 비디오 파일을 받아 딥페이크 여부를 분석하고 결과를 반환하는 딥페이크 탐지 API가 핵심입니다. Hugging Face Transformers를 활용한 Vision Transformer 모델 앙상블 추론을 통해 딥페이크를 탐지하며, OpenCV를 활용한 영상 프레임 추출 및 전처리, OpenCV Haar Cascade를 활용한 얼굴 감지 및 프레임 필터링을 수행합니다. 프레임별 분석 결과를 시간 구간별 세그먼트로 그룹화하여 타임라인을 생성하고, 분석 결과를 JSON 형식으로 반환합니다.

프론트엔드 측면에서는 카메라를 통한 직접 녹화 또는 앨범에서 영상 선택 기능이 필수입니다. 백엔드 API로 영상 파일을 전송하고, 딥페이크 확률 및 타임라인 정보를 시각화하여 표시하며, 직관적인 UI/UX를 제공합니다.

### Should: UX 개선 기능
백엔드에서는 영상 길이에 따라 최적의 프레임 추출 간격을 자동으로 조정하는 동적 프레임 샘플링 기능을 제공합니다. 5초 이하 영상은 0.5초 간격, 10초 이하는 1초 간격, 30초 이하는 2초 간격, 30초 초과는 3초 간격으로 설정하여 짧은 영상은 상세하게 분석하고 긴 영상은 효율적으로 처리합니다. 정적 화면의 오탐을 방지하기 위한 confidence 보정 알고리즘을 적용하며, ProcessPoolExecutor를 활용한 멀티프로세싱으로 분석 속도를 향상시킵니다. JSONL, CSV, JSON 형식의 데이터셋을 자동 변환하여 다운로드할 수 있으며, Firebase를 통한 영상 저장 및 관리 기능을 제공합니다.

프론트엔드에서는 Android MediaProjection API를 활용한 실시간 스크린 레코딩 기능이 핵심입니다. 앱 전환 없이 현재 화면을 녹화할 수 있으며, 15초 자동 종료 기능과 백그라운드 Foreground Service 실행을 지원합니다. 분석 중 모달 및 로딩 인디케이터를 통해 진행 상태를 표시하며, 타임라인 세그먼트를 그래프로 시각화합니다. 모바일 브라우저를 통한 파일 다운로드 기능과 네트워크 오류, 분석 실패 등 에러 상황을 처리합니다.

### Could: 추가 가치 기능
향후 추가할 수 있는 기능으로는 Whisper를 활용한 음성 인식 및 음성 기반 딥페이크 탐지, 대용량 영상 처리를 위한 비동기 분석 큐 시스템, 여러 영상을 한 번에 분석하는 배치 API 등이 있습니다. 여러 모델 버전 관리 및 A/B 테스트 기능, PDF/Excel 형식의 상세 분석 리포트 생성 기능도 포함됩니다.

프론트엔드에서는 분석 기록 저장 및 조회 기능, 분석 결과를 소셜 미디어에 공유하는 기능, 딥페이크로 판단된 영상을 플랫폼에 신고하는 기능, 사용자 간 분석 결과 공유 및 토론을 위한 커뮤니티 기능 등이 있습니다. 특정 확률 이상의 딥페이크 탐지 시 알림을 제공하는 알림 시스템, 다크 테마 지원, 여러 언어 지원 등도 고려할 수 있습니다.

## 사용자 시나리오

### 플로팅 위젯 사용 흐름
가장 대표적인 사용 시나리오는 사용자가 유튜브나 인스타그램에서 의심스러운 영상을 발견했을 때입니다. 사용자는 딥페이크 탐지 앱을 실행하고 HomeScreen에서 "위젯 제어"를 선택합니다. WidgetControlScreen에서 "Switch ON!" 버튼을 클릭하면 시스템 권한 요청 다이얼로그가 표시되고, 사용자가 권한을 허용하면 홈 화면으로 자동 이동합니다. 이때 플로팅 녹화 버튼이 화면에 표시됩니다.

사용자는 유튜브 앱으로 돌아가서 의심스러운 영상을 재생하고, 플로팅 버튼을 클릭하여 녹화를 시작합니다. MediaProjection 권한 요청 다이얼로그가 표시되면 허용하고, 백그라운드에서 화면 녹화가 시작됩니다. 최대 15초 동안 녹화되며, 15초가 경과하거나 사용자가 수동으로 종료할 수 있습니다. 녹화가 완료되면 "녹화 완료" 토스트 알림이 표시되고, "분석하시겠습니까?" 팝업이 나타납니다.

사용자가 "예"를 선택하면 RecordScreen으로 이동하며, "분석 중..." 모달이 표시됩니다. 백엔드 API로 영상이 전송되고, 딥러닝 모델로 분석이 수행됩니다. 분석 결과가 수신되면 "딥페이크 확률: 65%"와 같은 모달이 표시되고, ResultScreen으로 이동하여 상세 결과를 확인할 수 있습니다. 타임라인에서 딥페이크로 탐지된 구간을 확인하고, "데이터셋 다운로드" 버튼을 통해 연구용 데이터를 다운로드할 수 있습니다.

### 직접 녹화 사용 흐름
딥페이크 탐지 앱을 실행하고 HomeScreen에서 "녹화"를 선택하면 RecordScreen으로 이동합니다. 카메라를 통해 직접 녹화하거나 앨범에서 영상을 선택할 수 있으며, 녹화가 완료되면 "분석" 버튼을 클릭합니다. 백엔드로 영상이 업로드되고 분석이 수행되며, 분석 결과가 표시됩니다. 결과 화면에서 상세 정보를 확인할 수 있습니다.

### 데이터셋 생성 시나리오
연구자가 딥페이크 탐지 앱을 실행하고 여러 영상을 녹화 및 분석합니다. 각 분석 결과에서 "데이터셋 다운로드" 버튼을 클릭하면 JSONL, CSV, JSON 파일이 다운로드됩니다. 다운로드된 파일을 딥러닝 모델 학습에 활용할 수 있으며, 데이터셋을 통해 모델 성능 개선 연구를 수행할 수 있습니다.

### 사용자 여정 다이어그램
사용자 여정은 의심스러운 영상을 발견하는 것으로 시작됩니다. 앱을 실행하고 HomeScreen에서 위젯 제어 또는 직접 녹화/업로드를 선택할 수 있습니다. 위젯 제어를 선택하면 플로팅 위젯을 통해 녹화를 시작하고, 직접 녹화를 선택하면 카메라 또는 앨범에서 영상을 선택합니다. 두 경로 모두 영상 분석 단계로 이어지며, 백엔드 API를 통해 딥페이크 탐지가 수행됩니다. 결과 화면에서 결과 확인 및 공유, 또는 데이터셋 다운로드를 할 수 있습니다.

## 기술 요구사항 & 제약

### 기술 스택
프론트엔드는 React Native (Expo ~51.0.0)를 기반으로 하며, TypeScript로 개발되었습니다. React Navigation을 사용하여 Stack Navigator를 구현했고, React Hooks를 통해 상태 관리를 수행합니다. Android Native Modules는 Kotlin으로 작성되었으며, FloatingService에서 MediaProjection API와 Foreground Service를 활용합니다. 외부 라이브러리로는 expo-camera, expo-av, expo-file-system, axios, firebase 등이 사용되었습니다. Android 6.0 (API Level 23) 이상에서 동작합니다.

백엔드는 FastAPI 프레임워크를 사용하며, Python 3.8 이상에서 동작합니다. asyncio와 ProcessPoolExecutor를 활용한 비동기 및 병렬 처리를 지원하며, OpenCV를 통한 비디오 처리, PyTorch와 Hugging Face Transformers를 활용한 딥러닝 추론을 수행합니다. Deep-Fake-Detector-v2-Model과 google/vit-base-patch16-224-in21k 모델을 앙상블 방식으로 사용합니다. NumPy, PIL, pandas 등 데이터 처리 라이브러리를 사용하며, Uvicorn을 ASGI 서버로 사용합니다. Firebase Storage를 통해 영상 저장 및 관리를 수행합니다.

인프라 및 도구로는 Git과 GitHub를 통한 버전 관리, Gradle을 통한 Android 빌드, npm/yarn을 통한 Node.js 패키지 관리, Android Studio, VS Code/Cursor 등의 개발 환경이 사용됩니다.

### 제약 및 리스크
기술적 제약으로는 현재 Android만 지원하고 iOS는 미지원이라는 점이 있습니다. Android 6.0 이상이 필요하며, MediaProjection API는 Android 5.0 이상에서만 사용 가능합니다. 딥러닝 모델 추론은 CPU에서 수행 시 시간이 많이 소요되며, 영상 길이에 따라 분석 시간이 선형적으로 증가합니다. 모바일 기기의 제한된 메모리로 인해 큰 영상 처리가 어렵고, 딥러닝 모델 로딩 시 메모리 사용량이 증가합니다. 백엔드 API 통신이 필수적이므로 오프라인 환경에서 사용할 수 없으며, 동적 프레임 샘플링으로 인해 일부 프레임이 누락될 수 있습니다.

비용 관련 제약으로는 클라우드 서버 운영 비용, GPU 인스턴스 사용 시 비용 증가, Firebase Storage 비용, 모델 다운로드 비용 등이 있습니다. 사용자 증가 시 운영 비용이 급증할 수 있는 리스크가 있습니다.

보안 문제로는 사용자 영상 파일이 서버로 전송되는 점, Firebase 키 파일 관리 필요, MediaProjection 권한은 민감한 권한이라는 점, 현재 CORS가 모든 origin을 허용하는 개발용 설정이라는 점 등이 있습니다. 프로덕션 환경에서는 보안 취약점이 될 수 있습니다.

정확도 관련 제약으로는 딥러닝 모델의 본질적 한계로 인해 100% 정확도를 달성할 수 없다는 점, 새로운 딥페이크 기술에 대한 대응 지연, 오탐 및 미탐 가능성, 정적 화면 보정 알고리즘이 일부 경우 부정확할 수 있다는 점 등이 있습니다.

확장성 제약으로는 단일 서버에서 동시 처리 가능한 요청 수 제한, 현재 단일 모델 버전만 지원, 모델 업데이트 시 서비스 중단 필요 등의 문제가 있습니다.

## 향후 계획

### 최소 달성 목표 (MVP)
단기 목표로는 기본 기능을 완성하고 안정성을 개선하는 것입니다. 플로팅 위젯 기반 스크린 레코딩, 딥러닝 모델 앙상블 탐지, 타임라인 기반 결과 표시, 데이터셋 다운로드 기능은 이미 구현 완료되었습니다. 오류 처리 강화, 네트워크 오류 시 재시도 로직, 메모리 누수 방지, 성능 최적화를 통해 안정성을 개선하고, 분석 진행률 표시, 결과 화면 UI/UX 개선, 다크 모드 지원 등을 통해 사용자 경험을 개선할 예정입니다. 또한 사용자 가이드, API 문서화, 개발자 가이드 작성 등의 문서화 작업을 완성할 계획입니다.

중기 목표로는 iOS 지원을 추가하고 성능을 최적화하며 보안을 강화하고 분석 기능을 확장하는 것입니다. React Native iOS 빌드, iOS 화면 녹화 기능 구현, iOS 권한 처리를 통해 iOS를 지원하며, GPU 서버 활용으로 분석 속도를 향상시키고, 모델 경량화, 프레임 샘플링 알고리즘 개선을 통해 성능을 최적화합니다. API 인증 시스템 구현, 데이터 암호화, Firebase 보안 규칙 강화를 통해 보안을 강화하고, Whisper를 활용한 음성 분석 기능 추가, 오디오 기반 딥페이크 탐지, 멀티모달 분석(영상 + 음성)을 통해 분석 기능을 확장할 예정입니다.

### 발전 방향
장기 목표로는 AI/ML 개선, 플랫폼 확장, 기능 확장, 비즈니스 모델 구축, 연구 및 개발, 사회적 기여 등을 계획하고 있습니다. 더 정확한 딥페이크 탐지 모델 개발, 실시간 스트리밍 분석 지원, 다양한 딥페이크 기술 탐지(FaceSwap, DeepFaceLab 등), 자체 딥러닝 모델 학습 및 배포를 통해 AI/ML을 개선할 예정입니다. 브라우저에서 사용 가능한 웹 애플리케이션, Windows, macOS, Linux 지원 데스크톱 버전, Chrome, Firefox 확장 프로그램 개발, 타사 서비스에 API 제공 등을 통해 플랫폼을 확장할 계획입니다.

분석 기록 저장 및 조회, 소셜 미디어 공유, 플랫폼별 딥페이크 신고 연동, 사용자 간 정보 공유 및 토론 커뮤니티, 특정 확률 이상 탐지 시 알림 등의 기능을 확장할 예정입니다. Freemium 모델(기본 기능 무료, 고급 기능 유료), 미디어 기업 및 플랫폼 서비스에 기술 제공, 개발자 대상 API 구독 서비스, 고품질 딥페이크 데이터셋 판매 등을 통해 비즈니스 모델을 구축할 계획입니다. 일부 기능 오픈소스화, 딥페이크 탐지 연구 성과 공유, 대학 및 연구기관과 협력 연구, 연구용 데이터셋 공개 등을 통해 연구 및 개발에 기여하고, 미디어 리터러시 교육 콘텐츠 제공, 팩트체크 기관과 협력, 딥페이크 규제 정책 제안, 글로벌 딥페이크 대응 네트워크 참여 등을 통해 사회적으로 기여할 계획입니다.
